The "1-2 Sales" Daily Routine

Deep Work

2 Hours

  1. Review current master template status (SQLite/Convex sync, Reanimated UI).
  2. Improve one reusable module (e.g., offline lead capture, instant search).
  3. Test deployment speed: clone → brand → run in < 3 days target.
  4. Commit changes to private repo with clear notes.
Show Your Work

30 Minutes

  1. Choose one high-value feature (e.g., 0ms offline sync demo).
  2. Record 60-second screen capture (clean UI, voice-over or captions).
  3. Write short caption + industry tag.
  4. Post to LinkedIn, X, IG/TikTok — include procleaning.expo.app link.
Direct Outreach

1 Hour

  1. Find 5 local businesses (cleaning/roofing/real estate/solar) via Google Maps/Yelp.
  2. Personalize DM/email: mention their industry + pain point (slow apps, bad signal).
  3. Send message with procleaning.expo.app link + 2-min demo offer.
  4. Log contact in CRM (Obsidian note) with follow-up date.

Turn-Key Client Architecture

Phase 1

Ingestion (The Lead)

  1. Receive lead (DM, email, social reply) or send proactive outreach.
  2. Send procleaning.expo.app demo link immediately.
  3. Wait for response (“looks good, can it do X?”).
  4. Reply with branded 1-Page Sales PDF (anchors $1,500 price).
  5. Log interaction in Obsidian CRM note.
Phase 2

Commitment (The Close)

  1. Schedule 15-minute Fit Call (strategy, not sales pitch).
  2. Ask qualifying questions: employee count, iOS/Android mix, current tools.
  3. Send digital contract via DocuSign/HelloSign (define Turn-Key vs Custom scope).
  4. Generate Stripe Checkout link for 50% deposit ($750).
  5. Confirm payment → move note to “Active Projects” folder in Obsidian.
Phase 3

Execution (The Build)

  1. Send onboarding form link (logo vector, hex colors, services list, pricing).
  2. Clone private GitHub Field-Trades-Template into new private repo.
  3. Apply branding (logo, colors) + inject client backend credentials.
  4. Customize logic (3–5 days max) based on paid scope.
  5. Test offline/online sync, Reanimated UI, core features.
  6. Commit changes with clear messages; notify client of progress.
Phase 4

Deployment (The Handover)

  1. Generate TestFlight (iOS) / Expo Internal Testing (Android) link.
  2. Send beta link to client for review.
  3. Receive approval email/text: “Looks good to go”.
  4. Charge remaining 50% ($750) via Stripe.
  5. Publish to App Stores (your dev account or client’s).
  6. Activate $49/mo auto-pay (Managed SaaS only).
  7. Hand over admin credentials + basic usage doc.

End-to-End Client Flow

Fit & Filter
  1. Client inquires or responds to outreach.
  2. Request: GitHub username of tech lead + billing email for cloud.
  3. Determine path: Managed SaaS → standard Turn-Key | Buyout → asset transfer.
  4. Send appropriate contract template.
Asset Transfer (Buyout / HOA)
  1. Code Audit: Remove all personal API keys; convert to ENV variables.
  2. Send Stripe Invoice for Source Code Release Fee.
  3. Wait for full payment clearance.
  4. Transfer GitHub repo ownership to client’s GitHub Org.
  5. Migrate Convex team: invite client as Admin, swap billing card, remove self.
  6. Transfer Expo project to client Org (if applicable).
  7. Send signed Exit Agreement: “No ongoing responsibility post-transfer”.
  8. Archive project in Obsidian + mark complete.

Field Trades Template

Cold call script (service businesses)

White-label base: clone Field-Trades-Template per client. Demo: https://roof-cleaning-template.expo.app (Expo slug until renamed).

Mindset (before you dial)
You’re selling speed and clarity: a branded booking + job flow—not a science project. They don’t need to care how it’s built.
Win on the call: a 15-minute demo, or a one-pager + short Loom—not a closed deal.

Opening (permission + relevance)
You:
"Hi, is this [Name]? [Your name] — I’m reaching out to a few [city] [roof / lawn / plumbing / HVAC] shops. We set up a ready-made customer booking flow—schedule, deposit, reminders—so small crews spend less time on phone tag.
Have I caught you for 20 seconds, or is [later today / tomorrow morning] better?"

Bad time → book a specific callback. Good time → continue.

Bridge (one breath)
"Totally fine if it’s not a fit—I just want to see if how you take jobs today is anything like what we built for."

One-line value (use the line that matches their trade)
Roof / exterior: "Homeowners pick package, property size, and a time slot—you see everything in one queue instead of scattered DMs."
Lawn: "Recurring routes and seasonal add-ons without re-explaining pricing in texts every week."
Plumber / HVAC: "Emergency vs. standard intake, clearer dispatch, card on file so you’re not chasing checks after the job."

Template positioning (always say this once)
"It’s not a ground-up custom build—it’s a template you brand, so you’re live in days, not months, without agency-style pricing."

Qualification (pick 2–3; listen more than you talk)
"Where do most new jobs come in today—phone, Google, text, or a form?"
"When you’re on a job, what slips first—returning calls, quoting, or collecting deposits?"
"If fewer no-shows and money up front moved the needle, would 15 minutes on a screen share be reasonable?"

Transition to demo
You:
"Rather than describe it—I’ll show you what the homeowner sees and what you see on your side: appointments, jobs, estimates or invoicing if you use them.
Are you open [Option A: e.g. tomorrow 10] or [Option B: e.g. Thursday 2] for a quick share?"

If they hedge:
"No worries—I can record a 3-minute walkthrough for [their trade] and drop you a link. Best email or cell?"

Objections
They say                         You say
"We’re too small."              "That’s the sweet spot—one crew still pays for missed calls and no-shows."
"We have a website."            "Perfect—this sits under that: booking, payments, job flow. A lot of folks link out from the site or replace the contact form."
"Sounds expensive."             "Fair—on the demo I’ll show exactly what’s included so you can compare to hiring it out or building it yourself."
"I don’t have time."            "That’s why it’s 15 minutes—you’ll know fast if it’s wrong, and we stop."
"Just send info."               "Will do—one page plus a short video. Best email? And mostly residential or commercial so I don’t waste your time?"

Close
You:
"Great—I’ll [send the calendar link / hold Thursday 2]. You’ll walk through customer booking into your dashboard end to end. If it’s not a match, no hard feelings.
Thanks [Name]—talk [day/time]."

Voicemail (~25 seconds)
"Hi [Name], [Your name]. We help [trade] owners run bookings and deposits through a branded app—less phone tag, fewer no-shows. It’s a fast-launch template, not a custom dev project. I’ll text a one-liner and link from this number—thanks."

Text (after no pickup)
"[Name] — [Your name], booking flow for [trade] crews. Most of your jobs still phone/text? We bundle book + deposit + reminders in one place. 15-min demo if useful: [link or two time options]."

Conversion habits
• Lead with trade + city in the first sentence.
• Say "booking, deposits, reminders" before "app" or "template."
• One clear ask: demo, video, or email—never all three at once.
• After a demo: 3-bullet recap—what they saw, what pain it removes, "starts at" or range if you’re comfortable.
